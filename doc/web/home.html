What is Sereus?

Sereus (from Latin serere: to join, weave, connect) is a Web3 programming environment where connections (threads) are established between users by common consent and based on existing trust. Unlike traditional web identities that broadcast to the world, Sereus nodes join small, private subnets by invitation only.

Each thread is a shared SQL database, accessible only by its participants with role-based permissions. Identities are shared only among thread members, and credentials are exchanged out of band—eliminating the need for certificate authorities or central identity providers.

Designed for mobile-first deployment, Sereus runs efficiently on phones and other resource-constrained devices, bringing peer-to-peer capabilities to everyday mobile users.

Why Sereus?

  • Privacy by Design: No global identity directory. Your identity exists only within threads you've joined by consent.
  • Direct Peer Connections: Communicate directly via libp2p multiaddresses without central brokers or gatekeepers.
  • Shared SQL Databases: Each thread provides a familiar SQL interface, distributed across participants for resilience.
  • Developer Friendly: Build applications using standard SQL queries on top of a robust distributed foundation.
  • True Decentralization: No single point of control or failure. Even relay nodes can be privately operated.

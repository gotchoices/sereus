Get Started with Sereus

Development Path

Building on Sereus involves understanding its fundamental concepts, setting up your development environment, and creating your first thread-based application. Here's the path from zero to a working application.

Prerequisites

Before diving into Sereus development, you should have:

  • Linux/Unix environment: Most development happens on Linux; macOS works as well
  • SQL knowledge: Familiarity with SQL for defining schemas and querying data
  • Basic networking: Understanding of IP addresses, ports, and peer-to-peer concepts
  • Programming experience: Comfort with at least one programming language for building your application layer

Conceptual Foundation

Start by understanding the core concepts:

  1. Threads: Small, private subnets formed by invitation. Each thread is a shared SQL database among consenting participants.
  2. Cadres: Your cluster of devices that distribute and serve thread data. Think of it as your personal infrastructure.
  3. Roles and Permissions: Thread creators define who can do what using role-based access controls.
  4. libp2p Addressing: How nodes find and connect to each other without central servers.

Review the Overview and Architecture pages to build your mental model.

Development Steps

Step 1: Install the Sereus Stack

The Sereus stack consists of several components that work together:

  • Quereus: SQL query processor (gotchoices/quereus)
  • Optimystic: Storage engine with B-trees and block storage (gotchoices/Optimystic)
  • Fret: Distributed hash table (included with Optimystic)
  • libp2p: Peer-to-peer networking library (libp2p.io)
  • Bootstrap tools: Utilities for thread initialization and management

Note: Detailed installation documentation is in development. Check the GitHub repositories for current setup instructions.

Step 2: Initialize Your Development Environment

Set up a local development cadre:

  • Configure a local node with storage
  • Generate your node identity and keys
  • Set up libp2p multiaddresses for local development
  • Test that your node can store and retrieve data

Step 3: Create Your First Thread

Bootstrap a development thread:

  1. Define your schema: What tables and fields does your application need?
  2. Define roles: What permissions will different participants have?
  3. Initialize the thread: Use bootstrap tools to create the thread database
  4. Verify: Confirm you can query the thread database via SQL

Step 4: Connect a Second Node

Threads need at least two participants to be meaningful:

  • Set up a second node (can be on the same machine for development)
  • Generate an invitation from your first node
  • Have the second node join via the invitation
  • Verify both nodes can see the same thread database
  • Test distribution by writing from one node and reading from the other

Step 5: Build Your Application

Now you're ready to build application logic:

  • Choose your language: Any language with SQL support can work
  • Connect to Quereus: Your application issues SQL queries through the Quereus interface
  • Implement your logic: Write the code that reads and writes to the thread database
  • Handle events: React to changes in the thread (new messages, updated data, etc.)
  • Test thoroughly: Verify behavior with multiple nodes online and offline

Example Application: Simple Messaging

A minimal messaging app demonstrates the basics:

Schema

CREATE TABLE messages (
  id SERIAL PRIMARY KEY,
  sender TEXT NOT NULL,
  content TEXT NOT NULL,
  timestamp TIMESTAMP DEFAULT NOW()
);

CREATE TABLE participants (
  user_id TEXT PRIMARY KEY,
  display_name TEXT,
  role TEXT CHECK (role IN ('admin', 'member'))
);

Application Logic

  • Send message: INSERT INTO messages (sender, content) VALUES (?, ?)
  • Retrieve messages: SELECT * FROM messages ORDER BY timestamp DESC LIMIT 50
  • List participants: SELECT * FROM participants

The Sereus stack handles all distribution, storage, and network concerns. Your app just issues SQL.

Next Steps

Once you have a basic application working:

  • Explore advanced features: Multi-device cadres, complex role permissions, cross-thread references
  • Test at scale: Add more participants, simulate network partitions, measure performance
  • Build real applications: Choose a use case and build something useful
  • Join the community: Share your work, contribute back, help others learn

Learning Resources

  • GitHub Repositories: Source code and documentation for Quereus, Optimystic, and tools
  • libp2p Documentation: Learn about peer-to-peer networking at docs.libp2p.io
  • Community Forum: Connect with other developers (links coming soon)
  • Example Applications: Study reference implementations to understand patterns

Development Status

Sereus is under active development. Some components are mature while others are still evolving. The core concepts (threads, cadres, consent-based connections) are stable, but APIs and tooling continue to improve.

Early adopters should expect to work closely with the codebase and documentation, potentially contributing improvements as they learn.

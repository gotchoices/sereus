Technology Stack

An Integrated Foundation

Sereus Fabric brings together proven open-source technologies and purpose-built components, creating a cohesive programming environment for consent-based, peer-to-peer applications. Each layer of the stack addresses a specific challenge in distributed systems.

Core Components

Quereus: SQL Query Processor

Quereus provides the SQL interface for thread databases. It processes queries, manages transactions, and presents developers with a familiar relational database experienceâ€”even though the underlying data is distributed across multiple nodes.

  • Standard SQL syntax for queries and schema definitions
  • Transaction support for consistent updates
  • Query optimization for distributed storage
  • Role-based access control integration

GitHub: gotchoices/quereus

Optimystic: Storage Engine

Optimystic provides the underlying storage layer with B-trees, indexes, and block storage. It handles the complexities of sharding, distribution, and synchronization across cadre nodes.

  • B-tree data structures for efficient indexing
  • Block-level storage management
  • Distribution and consistency protocols
  • Coordination across distributed nodes

GitHub: gotchoices/Optimystic

Fret: Distributed Hash Table

Fret (part of the Optimystic distribution) implements a distributed hash table for content addressing and node discovery within threads. It enables efficient data location and retrieval across sharded storage.

  • Content-addressable storage
  • Efficient key-value lookups
  • Node discovery within thread networks
  • Integrated with Optimystic storage

Fret is distributed as part of Optimystic

libp2p: Peer-to-Peer Transport

libp2p is a mature, community-supported library that handles all aspects of peer-to-peer networking. Sereus leverages libp2p for establishing connections, NAT traversal, transport encryption, and more.

  • Multiple transport protocols (TCP, WebSockets, QUIC, etc.)
  • NAT traversal and hole-punching
  • Encrypted connections by default
  • Relay support for nodes behind restrictive networks
  • Multiaddress format for flexible addressing
  • Battle-tested in IPFS and other production systems

Learn more at libp2p.io

Bootstrap Tools

Sereus provides tooling to help developers initialize threads, configure cadres, and launch applications:

  • Thread initialization and schema setup
  • Cadre configuration and node management
  • Invitation generation and credential exchange
  • Development and debugging utilities
  • Deployment assistance for production environments

How the Stack Works Together

The components form a cohesive system:

  1. Application Layer: Your application issues SQL queries through Quereus, treating the thread database like any other SQL database.
  2. Query Processing: Quereus parses queries, applies role-based access controls, and translates operations into storage commands.
  3. Storage Layer: Optimystic manages the actual data storage using B-trees and block storage. It shards data across participating nodes and maintains consistency.
  4. Content Discovery: Fret's distributed hash table helps locate data blocks across the network and facilitates node discovery.
  5. Network Transport: libp2p establishes and maintains encrypted peer-to-peer connections between cadre nodes, handling all network-level concerns.
  6. Bootstrap & Management: Developer tools handle initialization, configuration, and operational management of threads and cadres.

Developer Benefits

This integrated stack provides significant advantages for developers:

  • Familiar Interfaces: Write standard SQL; the distribution is transparent.
  • Reduced Complexity: No need to implement your own distribution, consistency, or networking protocols.
  • Proven Components: Build on mature, tested libraries like libp2p rather than reinventing peer-to-peer networking.
  • Focus on Application Logic: Spend your time on features and user experience, not infrastructure plumbing.
  • Scalable Foundation: The architecture supports everything from two-person conversations to complex multi-party applications.

Use Case: Digital Currency

MyCHIPs: A Credit-Based Digital Medium of Exchange

The MyCHIPs digital medium of exchange project provides a system for managing credit relationships between individuals and businesses. Unlike cryptocurrency (which relies on tokens), MyCHIPs is based on credit tallies—records of mutual obligations between two parties.

The Traditional MyCHIPs Architecture

Historically, MyCHIPs has been based on two separate, individual databases kept by two parties to a single logical shared tally of credit entries. Each party maintains their own database and the system must ensure consensus between these two independent databases.

This architecture presented challenges:

  • Consensus complexity: Ensuring both parties' databases agree requires complex synchronization protocols
  • Conflict resolution: What happens when the databases disagree?
  • Application burden: The application layer must handle all storage, distribution, and consistency concerns
  • Availability: If one party's database is offline, the tally becomes inaccessible

MyCHIPs Retooled for Sereus Threads

MyCHIPs is being retooled to work on a Sereus thread. This represents a fundamental simplification: instead of two separate databases that must be kept in sync, there is one shared thread database accessible to both parties.

How It Works

When two parties establish a credit relationship:

  1. Thread creation: One party creates a tally thread and invites the other
  2. Shared database: Both parties' cadres share the same thread database
  3. Credit transactions: When goods or services change hands, the transaction is recorded in the shared database
  4. Automatic distribution: Changes are automatically distributed across both parties' cadres
  5. Consensus by design: There's only one authoritative record, eliminating sync issues

Benefits of Thread-Based Tallies

Simplified Consensus

With a shared database, consensus is inherent. Both parties see the same data because it's literally the same database, distributed across their cadres. There's no need for complex consensus protocols between separate databases.

Reduced Application Complexity

The application layer becomes much simpler. Instead of managing two databases, handling synchronization, and resolving conflicts, the MyCHIPs application just issues SQL queries to the thread database. The Sereus stack handles:

  • Storage and indexing (Optimystic)
  • Distribution across nodes (Optimystic + Fret)
  • Network transport (libp2p)
  • Consistency guarantees (thread database properties)

Improved Availability

Because the tally is distributed across both parties' cadres, it remains available even if some nodes are offline. If one party has multiple devices in their cadre, the tally is accessible from any of them.

Better Integration

Thread-based tallies can more easily integrate with other Sereus applications. For example:

  • A marketplace thread could reference tally threads for payment
  • Escrow services could participate in tally threads as third parties
  • Credit ratings could be computed from tally thread data

Credit vs. Token-Based Currency

MyCHIPs differs from cryptocurrencies like Bitcoin in fundamental ways:

Credit-Based

MyCHIPs tallies represent credit relationships: "Alice owes Bob 100 CHIPs" or "Bob owes Alice 50 CHIPs." This is more like traditional banking and how most commerce actually works.

  • No mining or proof-of-work
  • No global blockchain
  • Credit based on trust relationships, not scarcity
  • Can represent real economic value (goods/services exchanged)

Relationship-Focused

Each tally exists in a thread between specific parties. This creates natural privacy—your financial relationships aren't broadcast to the entire network.

Scalable

Because tallies are between pairs (or small groups) rather than on a global ledger, the system scales naturally. More users mean more tallies, but each tally is independent.

Technical Implementation

A MyCHIPs tally thread includes SQL tables for:

  • Tally record: Basic information about the credit relationship
  • Transactions (chits): Individual credit/debit entries
  • Balance: Current state of the tally (who owes whom how much)
  • Limits: Maximum credit each party will extend
  • Terms: Interest rates, settlement terms, etc.
  • Lifts: Multi-party settlements that net out circular debts

Role-based permissions ensure:

  • Both parties can read all tally data
  • Transactions require consent of both parties
  • Unilateral limits on how much credit each extends
  • Audit trail of all changes

Multi-Hop Payments and Lifts

One of MyCHIPs' powerful features is the ability to make payments through chains of tallies:

  • Alice wants to pay Carol but doesn't have a direct tally with her
  • Alice has a tally with Bob, and Bob has a tally with Carol
  • A payment can "hop" through: Alice credits Bob, Bob credits Carol
  • When circular debts exist (Alice→Bob→Carol→Alice), they can be "lifted" (netted out)

On Sereus, these operations can involve temporary multi-party threads or orchestration threads that coordinate across multiple bilateral tallies.

Advantages Over Cryptocurrency

  • No mining: No energy waste on proof-of-work
  • Private transactions: Tallies aren't publicly visible
  • Real value: Credit based on actual exchange of goods/services
  • Flexible credit: Parties set their own credit terms and limits
  • Scalable: No global blockchain bottleneck
  • Familiar model: Based on credit relationships people understand
  • Integrated: Can work alongside other thread-based applications

Advantages Over Traditional Banking

  • Peer-to-peer: No bank or central authority required
  • User-controlled: You own your tally data, not a bank
  • Low/no fees: No transaction fees extracted by intermediaries
  • Instant settlement: No clearing house delays
  • Transparent terms: All tally terms are explicit and visible to both parties
  • Inclusive: Anyone can participate; no credit score or bank account required

Current Status

The retooling of MyCHIPs for Sereus threads is an active development effort. The goal is to leverage Sereus's thread infrastructure to dramatically simplify MyCHIPs' application layer while maintaining all of its economic and privacy properties.
